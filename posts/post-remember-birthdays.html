Building a Birthday Reminder App with Spring Boot & React

By Keshav Callychurn

RememberBirthdays started as a simple itch to scratch — I kept forgetting friends' and family's birthdays. What began as a weekend project turned into a full-stack application I'm genuinely proud of. Here's how I built it.

The Tech Stack

I went with Java Spring Boot on the backend paired with PostgreSQL for persistence. The frontend uses React with TypeScript and Material UI. Everything runs in Docker containers with CI/CD via GitHub Actions.

Designing the API

The core entities are simple: a User and a Birthday. Each birthday has a name, date, and category. Standard CRUD endpoints:

GET    /api/birthdays
POST   /api/birthdays
PUT    /api/birthdays/{id}
DELETE /api/birthdays/{id}

Authentication uses JWT tokens. Spring Security intercepts each request and validates the token before allowing access to protected routes.

The Frontend

The React app has a dashboard showing upcoming birthdays, a form to add/edit entries, and a settings page. I used React Context for global auth state and React Query for data fetching and caching. TypeScript caught data shape mismatches at compile time rather than runtime — worth every keystroke.

Docker & Deployment

Each service runs in its own Docker container. A docker-compose.yml spins everything up locally. Frontend is on Netlify, backend on Render.

Lessons Learned

The biggest lesson: start with a simple working version, then add complexity. I over-engineered the auth system early on when a simpler setup would have been fine for the MVP. CORS, Flyway migrations, and environment variable management across environments were all things I hadn't dealt with before — each one a small win.
